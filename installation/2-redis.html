2. Redis¶

Installation¶

Redis is an in-memory key-value store which MrMap employs for caching and queuing. This section entails the installation and configuration of a local Redis instance. If you already have a Redis service in place, skip to the next section.

Note

MrMap require Redis v4.0 or higher. If your distribution does not offer a recent enough release, you will need to build Redis from source. Please see the Redis installation documentation for further details.

Debian¶

$ apt install -y redis-server

You may wish to modify the Redis configuration at /etc/redis.conf or /etc/redis/redis.conf, however in most cases the default configuration is sufficient.

Verify Service Status¶

Use the redis-cli utility to ensure the Redis service is functional:

$ redis-cli ping
PONG
