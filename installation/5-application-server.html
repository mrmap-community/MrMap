5. Application server¶

wsgi application server¶

We use gunicorn server for serving the wsgi application.

Note

You can also use other wsgi application servers such as uWSGI .

  1. If you use a virtualenv, activate it before installing gunicorn
$ source /opt/mrmap/venv/bin/activate
(venv) $ python -m pip install gunicorn

Configure¶

1. Adjust if needed and copy the config files to their destination: We need a service definition for gunicorn

# copy gunicorn systemd config (service file)
$ cp -a /opt/mrmap/install/confs/mrmap_gunicorn_service /etc/systemd/system/gunicorn.service
  1. Activate and start gunicorn service
$ systemctl enable gunicorn
$ systemctl start gunicorn

Verify Service Status¶

  1. Check the output of
$ systemctl status gunicorn

asgi application server¶

We use uvicorn server for serving the asgi application.

Note

You can also use other asgi application servers such as daphne .

  1. If you use a virtualenv, activate it before installing gunicorn
$ source /opt/mrmap/venv/bin/activate
(venv) $ python -m pip install uvicorn
(venv) $ python -m pip install websockets

Configure¶

2. Adjust if needed and copy the config files to their destination: We need a service definition for uvicorn

# copy uvicorn systemd config (service file)
$ cp -a /opt/mrmap/install/confs/mrmap_uvicorn_service /etc/systemd/system/uvicorn.service
  1. Activate and start uvicorn service
$ systemctl enable uvicorn
$ systemctl start uvicorn

Verify Service Status¶

  1. Check the output of
$ systemctl status uvicorn
